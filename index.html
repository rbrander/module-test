HTML Module test

Since Chrome v60, we have had the ability to use ES modules in HTML. This means large javascript applications can easily break up the code into smaller modules which can be loaded independantly. Simiarly to babel, when using modules, only one instance will be loaded if multiple imports are found.

In this test, we'll be using relative paths to reference the modules, which means we'll need a web server to provide CORS support, otherwise the browser will deny the request as the origin will be null
